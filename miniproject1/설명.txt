1.	Java 파일에 있는 폴더 중 util폴더에 있는 Util파일은 전부 보안에 관한 기능입니다.

	1)	MD5 매서드는 입력된 문자를 MD5형식의 해시 코드로 변환해서 보안성 높이는 용도입니다.
		SHA3-?을 사용하거나 랜덤으로 salt를 붙여 사용하거나 pepper를 사용하거나 bcrypt,argon2를 사용가능하지만
		SHA3-?,bcrtpt,argon2 는 들어가는 해시코드가 커서 varchar2를 BLOB나 CLOB으로 바꿔야하는 번거로움이 있어서 제외했고
		salt는 간단하게 살짝 더 넣으면 되나 slat도 깊게 들어가면 제대로 배워야하고
		pepper는 salt와 비밀번호를 따로따로 관리하여 붙여야한다는 번거로움이 있습니다.
		MD5방식은 제일 보안성이 떨어진다고 얘기되는 간단하게 만들수 있는 암호화방식입니다.
		
		* 실제로 서버를 열어서 운영하는 것이 아니라 암호화했다 라는 의미로 넣은 겁니다.
		넣고 싶어서 넣은 기능이므로 필요없다고 생각하시면 삭제해도 무방합니다.(비밀번호에 관한 메서드 수정 필요)
		
		암호화 해시생성기 사이트 : https://toolsyep.com/ko/haesi-saengseonggi/sha3-512/

	2)	HTML 이스케이프 처리는 XSS공격을 막기 위한 메서드입니다.
		HTML 코드로 blog를 공격하는 것을 막기 위한 보안처리이므로
		
		* 실제로 서버를 열어서 운영하는 것이 아니라 사실 의미가 없습니다.
		넣고 싶어서 넣은 기능이므로 필요없다고 생각하시면 삭제해도 무방합니다.(HTML이스케이프 시킨 메서드(입력,수정) 수정 필요)
	
	3)	이전 버전에 있는 Util 파일에 있던

		현재는 사용하는데에 혼선이 올 수 있어서 삭제했습니다.
		
		
2. 	cookie 에 대한 설명
	
	쿠키(인터넷 쿠키)는 사용자 이름과 비밀번호 등과 같은 작은 데이터 조각이 있는 텍스트 파일로, 사용자가 네트워크를 이용할 때 사용자의 컴퓨터를 식별하기 위해 사용
	Cookie cookie = new Cookie("user", value);
	쿠키를 선언하고 쿠키 이름을 "user" 로 지정 하고 그 값에 value 를 넣습니다.
	value에 "" 이 있으면 쿠키 삭제입니다.


3.	로그인 확인 메서드 혹은 관리자 확인 메서드 추가 사용방법
	
	// 자바빈을 사용 선언
	// 기능 확장 및 코드 재사용을 위함
	<jsp:useBean id="util" class="util.Util"></jsp:useBean>
	
	
	선언을 먼저 한 후
	로그인을 확인하는 메서드를 실행
	
	// 로그인 확인
	if (!Util.isLogIn(request)) {
	// 로그인이 되어 있지 않을 경우
    response.sendRedirect("mainpage2(login_before).jsp");
    }
    
    로그인이 안 되어 있을때 보낼 페이지는 ""수정하신 후 사용하시면 됩니다.
    여기까지만 작성해주시고 관리자라는 것을 확인을 해야하는 경우에
    
	else {
		// 세션에서 정보 가져오기
		MemberVo mv = (MemberVo) session.getAttribute("member");
		// 관리자일 경우
		int m_type = mv.getM_type();
		if (m_type == 2) {
			response.sendRedirect("../admin/mainpage(admin).jsp"); // 관리자인 경우 관리자 페이지로 리다이렉트
        }
	}
	
	을 추가적으로 선언해주시면 됩니다. 이 부분은 관리자 페이지에만 필요한 코드입니다.
	보통 로그인만 했으냐에 대해서 확인하는 메서드만 사용할 경우 자바빈을 사용할 필요는 없습니다.
	하지만 기능을 넣었으니 편하게 사용하시라고 사용 방법을 올렸습니다.
	관리자를 추가적으로 확인하기 위해서 코드 재사용을 위해 메서드를 자바빈을 이용한 것인데 
	jsp파일에 일일히 로그인 확인 메서드를 작성하시는 것보다 그냥 기능을 가져오는 것이 나을 것이라 생각합니다.
